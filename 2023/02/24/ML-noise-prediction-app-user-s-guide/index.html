WNPS user's guide

This is a brief user’s guide of the Wind Noise Prediction Software (WNPS). This project aimed to predict the wind noise propagation paths of the holes & gaps and provide guidance for the development of wind noise performance on the early stage. Machine learning and data science techniques are adopted to analysis former experimental data and conduct the prediction. Refer to the project developing report for all the details.

1. Introduction

This section gives an overview of Wind Noise Prediction Software (WNPS), its purpose, features, and benefits. It also provides some background information on the project development process and the intended audience of the guide. A glossary part is followed including terms or acronyms that could help users to understand the technical aspects of WNPS better.

1.1. Overview

WNPS is a cross-platform software product that predicts the wind noise propagation paths of the holes and gaps in a vehicle and provides guidance for the development of wind noise performance on the early stage. Wind noise is a type of noise that is generated by air flow around a vehicle and can affect its acoustic comfort and quality[1] [2]. WNPS uses machine learning and data science techniques to analyze former experimental data and conduct the prediction[3].

WNPS has the following features and benefits:

  • It can read the basic data of the experiment vehicle, such as vehicle type, height, width, length, configuration and status[3]
  • It can output a relative ML predicted AI (Articulation Index) value that indicates the intelligibility of speech in noisy environments[4]
  • It can print a line chart that shows how different experiment statuses affect the AI value[3]
  • It can help engineers to optimize the design of vehicles for better wind noise performance

The intended audience of this guide are engineers who are involved in vehicle design, testing and evaluation. This guide assumes that they have some basic knowledge of wind noise, machine learning and data science.

3. Getting started

This section helps you to get familiar with WNPS interface, menus, icons, buttons and other elements. It also include a tutorial or a walkthrough of a common task/scenario using WNPS.

3.1. Graphical User Interface (GUI)

The WNPS interface is designed to be simple and user-friendly, with three main parts: the menu bar, input panel, and output panel.

Part 1: the Menu Bar

The menu bar is located at the top of the WNPS window and provides access to various functions and settings. It contains the following options:

  • File: Allows you to open, save, and export files. It also allows you to the start the prediction process.
  • Plotting: Allows you to customize the plotting process and the appearance of output panel.

Part 2: the Input Panel

The input panel is located at the bottom half of the WNPS window and is where you can manually enter or select the basic data of your experiment vehicle. The input panel is divided into different sections, such as vehicle type, height, width, length, configuration, and status. Here is a brief description of the sections:

  • Vehicle basic information: Allows you to select the type of vehicle you are testing, and its dimension.
  • Configuration: Allows you to select the configuration of the vehicle.
  • Status: Allows you to enter the status of the experiment.

At the bottom of the input panel, there are two buttons: predict and save data. The predict button starts the prediction process, and the save data button allows you to save the input data as a CSV file.

Part 3: the Output Panel

The output panel is located at the upper half of the WNPS window and displays your prediction results as a point and/or as a line chart. The output panel is divided into two parts: the toolbar and the chart area.

  • Toolbar: Allows you to change the chart settings, such as zoom in/out, pan, and switch between AI point and line chart.
  • Chart area: Displays your prediction results as an AI point and/or as a line chart. The line chart shows how different experiment statuses affect the AI value. The relative (instead of the absolute) AI value is plotted indicates the AI gain/loss compared with the last status.

3.2. Tutorial

This tutorial will guide you through the process of using WNPS to predict wind noise performance and optimize vehicle design. By following these steps, you will be able to input your data, run the prediction process, view your results, and save your data and charts.

Step 1: Enter or select your input data

There are two methods for inputting your data into WNPS. The first method is manual input. This involves typing or choosing your input values from drop-down lists in each field under Input Data section in the input panel. Details will be explained in the manual input section.

The second method is data file input. To use this method, click on the Predict_from_file button under File section in the menu bar, and select your EXCEL file that contains your input data. The file should have one row per experiment status with columns corresponding to each input field. An example input file is shown in the file input section.

Step 2: Click on Predict button

Once you have entered or selected your input data, click on the Predict button at the bottom left corner of the input panel to start the prediction process.

Step 3: View your prediction results

After the prediction process is complete, you will be able to view your results in the output panel. Depending on your input data, you will see a new point or a line chart. The line chart shows how different experiment statuses affect the AI value. The relative (instead of the absolute) AI value is plotted, which indicates the AI gain/loss compared with the last status.

Step 4: Add a new line

If you want to add a new line to your input data, press the new_line button under the Plotting section on the menu bar, and then go back to step 1 to enter more data.

Step 5: Save your results

You can save your result as a data (CSV) file by clicking on the save data button at the bottom right corner of the input panel. You can also save the line chart by clicking on the save_image button under the plotting section on the menu bar, or clicking on the icon: save button on the output panel toolbar.

4. Using WNPS

This section explains how to perform various functions and operations using WNPS in detail. It provides tips, tricks and best practices for using WNPS effectively.

4.1 Manual input

The input panel of the Wind Noise Prediction Software (WNPS) is a crucial component of the software that enables engineers to perform virtual experiments to predict wind noise propagation paths in a vehicle. It contains three distinct subsections that enable users to input critical information about the vehicle being tested, wind-noise related configurations, and experiment status.

The first subsection, Basic Info, enables users to input information about the vehicle being tested, such as its type, dimensions, and other basic data. The Wind-Noise Related Configurations subsection allows the user to specify design details that could affect the vehicle's wind noise performance, such as hole size and shape, gap width, and vehicle configuration. The Experiment Status subsection allows the user to toggle between different settings for each virtual experiment.

The input panel is designed to make it easy for engineers to perform a series of experiments on a vehicle quickly. If they are performing a series of experiments at a single velocity to generate one AI line, they can set the first two subsections only once. They can then toggle the checkboxes under the third subsection several times to correspond with each virtual experiment, saving time and simplifying the process.

The Experiment Status subsection of the input panel is particularly useful because it enables engineers to quickly toggle between different settings for each virtual experiment. This means they can test multiple configurations in a short amount of time, without manually inputting all of the information again.

  • The wind noise related configurations section (Columns F to P) contains information about rearview mirror installation method, water retaining strip, front window glass type and thickness, sealing strips for various parts of the car (A-pillar, B-pillar, C-pillar and rear cover), whether or not a roof rack is present etc. For True/False configurations, 1 indicates that the design feature is present, while 0 indicates that it is not.

  • The experiment status section (Columns R to AH ) specifies the removal of tape from different parts of the vehicle during the virtual experiment. With 1 means yes and 0 means no (shaded as green and white for illustration). With 1 representing yes and 0 representing no, each experiment represents a different configuration, and the removal of tapes allows engineers to test the effects of different design choices on wind noise performance.

  • It is important to note that the Delta AI Value column does not affect the result and can be left at 0 for convenience. After prediction, it is possible to overwrite this column with the predicted value.

For a normal series of virtual experiments, the basic data and wind noise configurations for one vehicle remain unchanged, and the data of columns A to Q remain the same. In the experiment status section (Columns R to AH), the first experiment always tapes all parts of the vehicle, and with each following experiment, one new tape is removed until the last experiment removed all the tapes, representing the baseline status.

4. Machine Learning Techniques used in WNPS

This section briefly explains how the machine learning and data science techniques used by WNPS to analyze former experimental data and conduct wind noise prediction. It also includes a introduction of the different models and algorithms used by WNPS and how they are applied to the data input by the engineer.

Machine learning is a subset of artificial intelligence that involves the development of algorithms and statistical models that can learn from data and make predictions based on patterns. Machine learning has gained widespread popularity in recent years due to its ability to automate data analysis and prediction tasks, which were previously done manually.

4.1 Data Science in WNPS

The first step in using machine learning for wind noise prediction is to collect and preprocess the data. WNPS has a database of experimental data on wind noise generated by different vehicles under various environmental conditions. The data collected includes the vehicle type, wind speed, and other relevant parameters.

The success of wind noise prediction highly depends on the accuracy and quality of input data. As such, data scientists in the WNPS team utilize various data cleansing and analysis techniques to ensure the input data is reliable and consistent.

One of the data cleansing methods commonly used in WNPS is correlation analysis. This method helps to identify any correlations or relationships between different variables in the input data. For example, correlation analysis can reveal if there is a relationship between wind speed and wind noise level. If a strong correlation is found, the WNPS model can take it into account and make more accurate predictions.

Another method used in data cleansing is outlier detection. Outliers are data points that deviate significantly from the rest of the data, and they can have a significant impact on the accuracy of prediction models. Therefore, WNPS data scientists use various techniques to detect and remove outliers from the input data, such as using statistical methods like Z-score and box plots.

Additionally, data normalization and standardization are also used in WNPS. Normalization involves scaling the data to a common range to eliminate the impact of different units or scales. Standardization involves converting the data to a standard normal distribution with a mean of zero and a standard deviation of one. These methods ensure that the input data is consistent and can be analyzed accurately.

4.2 Machine Learning Models in WNPS

After the data has been collected and preprocessed, machine learning algorithms are applied to analyze and predict wind noise propagation paths. WNPS uses supervised learning algorithms, which require labeled data to train statistical models that can predict future wind noise propagation paths. The algorithms used include linear regression, decision trees, and neural networks.

Linear regression is a simple machine learning algorithm that finds the relationship between two variables by fitting a linear equation to the data. In WNPS, linear regression is used to model the relationship between wind speed, wind direction, and AI values, which are used to predict wind noise propagation paths.

Decision trees are another type of machine learning algorithm used in WNPS. Decision trees use a tree-like model to make decisions based on a series of conditions. In WNPS, decision trees are used to model the relationship between various input parameters and the wind noise output.

Finally, neural networks are a more complex type of machine learning algorithm used in WNPS. Neural networks are inspired by the structure and function of the human brain and consist of interconnected nodes that process information. In WNPS, neural networks are used to model the complex relationship between input parameters and wind noise propagation paths.

4.3 Validation and Optimization

After the machine learning models have been developed, they are validated and optimized to ensure that they provide accurate and reliable predictions. This is done by comparing the predicted wind noise values with the actual wind noise generated by the vehicle. The models are then optimized by adjusting the input parameters to minimize the difference between predicted and actual wind noise values.

Metrics such as accuracy, precision, recall, and F1-score are used to measure the performance of our models. In addition, techniques such as cross-validation and hold-out validation are used to estimate the generalization performance of our models. This will help us determine how well our models are likely to perform on new, unseen data.

In conclusion, machine learning is a powerful tool that can be used to predict wind noise propagation paths in vehicles. WNPS uses a combination of supervised learning algorithms, including linear regression, decision trees, and neural networks, to analyze former experimental data and predict wind noise propagation paths. These models are then validated and optimized to ensure that they provide accurate and reliable predictions.

5.2. FAQs

Some FAQs (frequently asked questions) about WNPS are:

Q: What is wind noise?

A: Wind noise is a type of noise that is generated by air flow around a vehicle and can affect its acoustic comfort and quality .

Q: What is AI (Articulation Index)?

A: AI (Articulation Index) is a measure of speech intelligibility in noisy environments. It ranges from 0 to 1, where higher values indicate better speech intelligibility.

Q: How does machine learning work in WNPS?
